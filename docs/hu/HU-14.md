# HU-14: Marcar Anuncio como Vendido

## Objetivo

- Actor: vendedor autenticado
- Capacidad: cerrar una venta asociando el anuncio a un comprador
- Resultado esperado: generar una transaccion valida y habilitar calificaciones

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `PATCH` | `/api/v1/anuncios/:anuncio_id/vendido` | Marcar anuncio como vendido |

## Entrada

```json
{ "comprador_id": 42 }
```

## Flujo principal

1. validar `anuncio_id`
2. validar JWT y cuenta `ACTIVO`
3. verificar existencia del anuncio
4. verificar propiedad del anuncio
5. permitir la operacion solo si `estado = ACTIVO`
6. validar existencia y estado del comprador
7. impedir autocompra
8. verificar que el comprador haya contactado el anuncio
9. actualizar el anuncio a `VENDIDO`
10. registrar `comprador_id` y `vendido_at`
11. crear registro en `transacciones`

## Datos relevantes

### Tabla implicada

- `transacciones`

## Respuesta esperada

`200 OK` con datos del anuncio vendido y bloque de transaccion creada.

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | identificador o body invalido |
| `401` | sin JWT o token invalido |
| `403` | falta de propiedad, cuenta no activa o anuncio bloqueado |
| `404` | anuncio inexistente |
| `409` | autocompra o conflicto de estado |
| `422` | comprador invalido o sin contacto previo valido |

## Reglas

- `comprador_id` es obligatorio
- `vendido_at` lo define el backend
- el anuncio vendido deja de estar visible en feed y busquedas
- la venta creada habilita HU-15 y HU-16
