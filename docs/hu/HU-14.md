# HU-14: Marcar Anuncio como Vendido

**Como** vendedor autenticado
**Quiero** marcar mi anuncio como vendido e indicar quién fue el comprador
**Para** confirmar la transacción y habilitar el sistema de calificaciones.

## Endpoint

PATCH `/api/v1/anuncios/:anuncio_id/vendido`
🔒 JWT requerido

## Body

```json
{
  "comprador_id": 42
}
```

## Flujo (orden obligatorio)

1. Validar `anuncio_id` entero positivo (sino 400).
2. Verificar JWT válido + cuenta `ACTIVO` (sino 401/403).
3. Verificar anuncio existe (sino 404).
4. Verificar propietario: `usuario_id` del JWT == `anuncio.usuario_id` (sino 403).
5. Verificar `estado=ACTIVO` (BLOQUEADO → 403, INACTIVO → 409, ya VENDIDO → 409).
6. Verificar `comprador_id` existe en BD y cuenta `ACTIVO` (sino 422).
7. Verificar `comprador_id` ≠ `usuario_id` del JWT (sino 409).
8. Verificar que `comprador_id` contactó el anuncio vía HU-12 — existe registro en `contactos_log` (sino 422).
9. Ejecutar en **transacción SQLAlchemy**:
   - `UPDATE anuncios SET estado="VENDIDO", comprador_id=:comprador_id, vendido_at=NOW()`.
   - `INSERT INTO transacciones (anuncio_id, vendedor_id, comprador_id, created_at)`.
10. Log: `vendedor_id, comprador_id, anuncio_id, timestamp`.

## Respuesta

**200 OK**
```json
{
  "anuncio_id": 15,
  "titulo": "Ryzen 5 5600X casi nuevo",
  "estado": "VENDIDO",
  "vendido_at": "2026-06-20T20:00:00",
  "transaccion": {
    "id": 8,
    "vendedor_id": 7,
    "comprador_id": 42,
    "calificacion_vendedor_pendiente": true,
    "calificacion_comprador_pendiente": true
  }
}
```

## Errores

- 400 `anuncio_id` inválido / `comprador_id` ausente en body.
- 401 Sin JWT o token inválido.
- 403 No es propietario / cuenta no ACTIVO / anuncio BLOQUEADO.
- 404 Anuncio no existe.
- 409 Autocompra / anuncio ya VENDIDO / anuncio INACTIVO.
- 422 `comprador_id` no existe / cuenta no ACTIVO / comprador nunca contactó el anuncio.

## Esquema `transacciones`

id                            → PK
anuncio_id                    → FK anuncios
vendedor_id                   → FK usuarios
comprador_id                  → FK usuarios
calificacion_vendedor_pending → BOOLEAN default TRUE
calificacion_comprador_pending→ BOOLEAN default TRUE
created_at                    → TIMESTAMP

## Reglas

- `comprador_id` obligatorio — la venta debe asociarse a un comprador real registrado en la plataforma.
- El comprador debe haber contactado el anuncio vía HU-12 — verificado desde `contactos_log`.
- `vendido_at` asignado por el backend, nunca por el body.
- Anuncio `VENDIDO` desaparece del feed y búsquedas — invisible para terceros.
- Anuncio `VENDIDO` visible para vendedor y comprador en su historial (HU futura).
- Anuncio `VENDIDO` no puede editarse, reactivarse ni reportarse.
- `transacciones` es registro permanente — nunca se elimina.
- Ambos flags `calificacion_*_pending` inician en `TRUE` — HU-15 y HU-16 los cambian a `FALSE` al calificar.
- Una vez marcado como `VENDIDO` no hay marcha atrás desde este endpoint — requiere intervención admin.
- `reactivaciones_count` no se resetea — si el vendedor republicase desde HU-05 sería un anuncio nuevo.
