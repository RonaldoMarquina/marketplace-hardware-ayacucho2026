# HU-08: Maquina de Estados del Anuncio

## Objetivo

- Tipo: historia de referencia funcional y arquitectonica
- Alcance: definir estados validos del anuncio, transiciones permitidas y
  restricciones por rol
- Resultado esperado: unificar el comportamiento de cambio de estado en todo el
  sistema

## Naturaleza de la HU

Esta HU no define un endpoint propio. Su funcion es establecer las reglas
transversales que deben respetar las HUs que modifican el estado de un anuncio.

## Estados validos

| Estado | Visible en feed | Editable por vendedor | Contactable | Terminal para vendedor |
|--------|-----------------|-----------------------|-------------|------------------------|
| `ACTIVO` | Si | Si | Si | No |
| `INACTIVO` | No | Si | No | No |
| `VENDIDO` | No | No | No | Si |
| `BLOQUEADO` | No | No | No | Si |

## Transiciones permitidas

| Estado origen | Estado destino | Responsable | Referencia |
|---------------|----------------|-------------|------------|
| `ACTIVO` | `INACTIVO` | vendedor | HU-07 |
| `INACTIVO` | `ACTIVO` | vendedor | HU-07 |
| `ACTIVO` | `VENDIDO` | vendedor | HU-14 |
| `ACTIVO` | `BLOQUEADO` | administrador | HU-13 |
| `BLOQUEADO` | `ACTIVO` | administrador | HU-13 |
| `ACTIVO` | `INACTIVO` | administrador, por efecto de bloqueo de cuenta o gestion relacionada | HU-20 |

## Transiciones no permitidas

Las siguientes transiciones deben rechazarse con conflicto de estado:

- `INACTIVO -> VENDIDO`
- `VENDIDO -> ACTIVO`
- `VENDIDO -> INACTIVO`
- `VENDIDO -> BLOQUEADO` desde flujo del vendedor
- `BLOQUEADO -> VENDIDO`
- `ACTIVO -> ACTIVO`
- `INACTIVO -> INACTIVO`
- `VENDIDO -> VENDIDO`
- `BLOQUEADO -> BLOQUEADO`

## Reglas globales de estado

- no se usa borrado fisico del anuncio
- `ACTIVO` es el unico estado visible en feed y busquedas publicas
- `INACTIVO` oculta el anuncio del espacio publico, pero el propietario aun
  puede gestionarlo
- `VENDIDO` cierra el ciclo comercial del anuncio para el vendedor
- `BLOQUEADO` suspende completamente la visibilidad y operacion del anuncio
- `created_at` no debe modificarse durante cambios de estado
- `updated_at` debe actualizarse en cada transicion valida
- la media y la trazabilidad del anuncio se conservan en todos los estados

## Reglas de visibilidad por estado

| Estado | Feed y busqueda | Detalle publico | Panel del propietario | Admin |
|--------|------------------|-----------------|-----------------------|-------|
| `ACTIVO` | Visible | Visible | Visible | Visible |
| `INACTIVO` | No visible | No visible para terceros | Visible | Visible |
| `VENDIDO` | No visible | No visible para terceros | Visible en contexto historico | Visible |
| `BLOQUEADO` | No visible | No visible | Restringido segun flujo de negocio | Visible |

## Restricciones por rol

### Vendedor

- puede desactivar un anuncio `ACTIVO`
- puede reactivar un anuncio `INACTIVO`
- puede marcar como vendido un anuncio `ACTIVO`
- no puede modificar un anuncio `VENDIDO`
- no puede revertir manualmente un anuncio `BLOQUEADO`

### Administrador

- puede bloquear un anuncio `ACTIVO`
- puede desbloquear un anuncio `BLOQUEADO`
- puede consultar el estado real del anuncio en endpoints administrativos

## Politica de reactivacion

| Rol | Limite de anuncios activos | Limite de reactivaciones por anuncio |
|-----|-----------------------------|--------------------------------------|
| `USER_ESTANDAR` | 25 | 3 |
| `TIENDA_VERIFICADA` | Sin limite | 3 |

Reglas asociadas:

- `reactivaciones_count` se inicializa en `0` al crear el anuncio
- cada reactivacion valida incrementa `reactivaciones_count`
- al superar el limite, el anuncio no debe poder volver a `ACTIVO`
- reactivar no debe alterar la posicion historica del anuncio via `created_at`

## Exposicion del campo `estado`

| Contexto | Exposicion |
|----------|------------|
| Feed publico | No se expone; se filtra por consulta |
| Busqueda publica | No se expone; se filtra por consulta |
| Detalle publico | No se expone a terceros |
| Panel del usuario | Si, por tratarse de vista privada |
| Endpoints de transicion | Si, como confirmacion de la accion |
| Endpoints administrativos | Si, por necesidad operativa |

## Trazabilidad

Toda transicion relevante debe dejar evidencia en el mecanismo de auditoria que
corresponda segun el modulo administrativo o funcional involucrado.
