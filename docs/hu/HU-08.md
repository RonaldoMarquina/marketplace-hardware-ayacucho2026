# HU-08: Desactivación / Borrado Lógico de Anuncio

**Como** vendedor autenticado  
**Quiero** marcar anuncios como vendido o desactivarlos  
**Para** que dejen de aparecer en el feed.

## Endpoints

PATCH `/api/v1/anuncios/:anuncio_id/vendido`  
PATCH `/api/v1/anuncios/:anuncio_id/desactivar`  
PATCH `/api/v1/anuncios/:anuncio_id/reactivar`  
🔒 JWT requerido

## Transiciones permitidas

```
ACTIVO   → VENDIDO   (/vendido)
ACTIVO   → INACTIVO  (/desactivar)
INACTIVO → ACTIVO    (/reactivar)
VENDIDO  → [terminal, ninguna]
BLOQUEADO→ [ninguna para el vendedor]
```

## Flujo (orden obligatorio)

1. Validar JWT.
2. Verificar anuncio existe (sino 404).
3. Verificar propietario (sino 403).
4. Verificar transición válida según estado actual (sino 409/403).
5. En `/reactivar`: verificar que usuario tenga < 10 anuncios ACTIVOS (sino 403).
6. UPDATE estado + `updated_at=NOW()`.
7. Log: usuario_id, anuncio_id, estado_anterior, estado_nuevo, timestamp.

## Respuesta

**200 OK**
```json
{
  "id": 15,
  "estado": "VENDIDO",
  "mensaje": "Tu anuncio ha sido marcado como vendido exitosamente.",
  "updated_at": "2026-01-01T00:00:00"
}
```

## Errores

- 403 No es propietario / BLOQUEADO / límite 10 anuncios al reactivar.
- 404 Anuncio no existe.
- 409 Transición inválida (ej. VENDIDO→VENDIDO).

## Reglas

- Nunca ejecutar DELETE, siempre borrado lógico.
- VENDIDO e INACTIVO no aparecen en feed ni búsquedas.
- VENDIDO es estado terminal para todos los roles.
- Historial accesible en `GET /api/v1/anuncios/mis-anuncios?estado=VENDIDO`.
