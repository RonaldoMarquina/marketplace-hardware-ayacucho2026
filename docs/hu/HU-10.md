# HU-10: Filtrado Tecnico de Anuncios

## Objetivo

- Actor: comprador con requerimientos tecnicos
- Capacidad: filtrar anuncios por atributos comerciales y especificaciones
- Resultado esperado: encontrar productos compatibles con mayor precision

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `GET` | `/api/v1/anuncios/buscar` | Buscar anuncios con filtros |

El endpoint es publico.

## Parametros de consulta

### Filtros base

- `categoria`
- `subcategoria`
- `condicion`
- `precio_min`
- `precio_max`
- `q`
- `order_by`
- `page`
- `limit`

### Filtros de especificaciones

Se admiten parametros con forma:

```text
specs[clave]=valor
```

Ejemplo:

```text
/api/v1/anuncios/buscar?subcategoria=Procesador&specs[socket]=AM4
```

## Validaciones funcionales

- `categoria` y `condicion` deben pertenecer a sus conjuntos validos
- `subcategoria` debe usar formato controlado
- `precio_min` y `precio_max` deben ser positivos
- si ambos existen, `precio_min <= precio_max`
- `q` debe respetar longitud minima
- `order_by` debe pertenecer a los valores permitidos
- cada clave de `specs[]` debe usar formato seguro
- maximo `10` filtros de especificaciones por solicitud

## Flujo principal

1. validar parametros de paginacion y orden
2. validar filtros comerciales
3. validar rango de precio
4. validar filtros `specs[]`
5. construir consulta base con anuncios `ACTIVO`
6. encadenar filtros recibidos con logica `AND`
7. aplicar orden solicitado
8. paginar resultados
9. unir informacion del vendedor
10. obtener `imagen_principal`
11. retornar resultados y bloque de `filtros_aplicados`

## Reglas de consulta

- todos los filtros son opcionales
- sin filtros, el comportamiento debe ser equivalente al feed principal
- la busqueda textual usa coincidencia controlada sobre titulo, descripcion y
  subcategoria
- las consultas deben usar parametros seguros, sin interpolacion directa
- las especificaciones se consultan dentro del JSON del anuncio

## Respuesta esperada

`200 OK` con:

- `data`
- `filtros_aplicados`
- `paginacion`

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | rango de precio invalido o busqueda textual insuficiente |
| `422` | filtros o claves de especificaciones invalidas |

## Reglas

- sin resultados se retorna `200` con `data: []`
- el endpoint no expone `estado`, `rol` ni campos privados del vendedor
- `rol` se transforma a `es_tienda_verificada`
- el filtrado debe mantenerse compatible con el esquema JSON de
  `especificaciones`
