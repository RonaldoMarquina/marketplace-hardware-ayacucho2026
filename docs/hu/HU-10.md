# HU-10: Filtrado Técnico de Anuncios

**Como** comprador con requerimientos técnicos  
**Quiero** filtrar por categoría, condición, precio y specs  
**Para** encontrar el componente compatible sin revisar uno por uno.

## Endpoint

GET `/api/v1/anuncios/buscar`  
🌐 Público — sin JWT

## Query params

| Param       | Validación                          |
|-------------|-------------------------------------|
| categoria   | ENUM válido                         |
| condicion   | ENUM válido                         |
| precio_min  | DECIMAL > 0                         |
| precio_max  | DECIMAL > 0                         |
| spec_key    | Alfanumérico + guion bajo, con spec_value |
| spec_value  | Requerido si spec_key presente      |
| q           | 2–100 chars, escapar `% _ \`       |
| page        | Entero > 0, default=1               |
| limit       | Entero 1–50, default=20             |

## Flujo

- Todos los filtros son opcionales, se encadenan con AND.
- Query base siempre incluye `WHERE estado='ACTIVO'`.
- Filtros adicionales según params recibidos:
  - `AND categoria = :categoria`
  - `AND condicion = :condicion`
  - `AND precio >= :precio_min AND precio <= :precio_max`
  - `AND JSON_UNQUOTE(JSON_EXTRACT(especificaciones, '$.:spec_key')) = :spec_value`
  - `AND (titulo LIKE :q OR descripcion LIKE :q)` (case-insensitive)
- OFFSET = (page - 1) * limit.
- Usar **prepared statements**, nunca interpolación de strings.
- `filtros_aplicados` solo lista params recibidos (sin nulls).

## Errores

- 400 `precio_min` > `precio_max`.
- 400 `spec_key` sin `spec_value` o viceversa.
- 422 `categoria` o `condicion` fuera de ENUM.
- 422 `spec_key` con caracteres no permitidos.
- 422 Intento de inyección SQL en params.

## Reglas

- Sin resultados → 200 con `data: []`, nunca 404.
- Sin filtros → comportamiento idéntico a HU-09.
- `spec_key` inexistente en JSON → `JSON_EXTRACT` retorna NULL → array vacío, no es error.
- `q` sanitizado: escapar `%`, `_`, `\` antes del LIKE.
- Búsqueda `q` con `COLLATE utf8mb4_unicode_ci` (case-insensitive).
- Paginación se calcula sobre resultados filtrados.
- Índice funcional en `JSON_EXTRACT(especificaciones, '$.socket')` si BD > 1000 registros.
