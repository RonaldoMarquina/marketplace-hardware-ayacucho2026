# HU-04: Autenticación (Login + JWT)

**Como** usuario verificado  
**Quiero** iniciar sesión con correo y contraseña  
**Para** recibir un JWT y acceder a funciones protegidas.

## Endpoint

POST `/api/v1/auth/login`

## Entrada

- correo
- password

## Flujo

Orden de validación:
1. Validar campos vacíos con Marshmallow.
2. Verificar correo existe en BD.
3. `bcrypt.check_password_hash(hash, password)`.
4. Verificar estado del usuario.
5. Generar JWT con Flask-JWT-Extended.

JWT payload: `{ sub, correo, rol, iat, exp: NOW()+8h }`  
JWT_SECRET en `.env`, nunca hardcodeado.

## Respuesta

**200 OK**
```json
{
  "token": "eyJ...",
  "rol": "USER_ESTANDAR",
  "nombre": "Juan Quispe",
  "expira_en": "8h"
}
```

## Errores

- 400 Campos vacíos.
- 401 Credenciales inválidas (mismo mensaje para correo inexistente y password incorrecto).
- 401 Token expirado / inválido (en endpoints protegidos).
- 403 Cuenta PENDIENTE_VERIFICACION o BLOQUEADA.

## Reglas

- Mensaje idéntico para correo no existe y password incorrecto (prevenir enumeración).
- No retornar password en ninguna respuesta.
- Rate limit: bloquear IP 15 min tras 5 intentos fallidos.
- Sin refresh token en este MVP.
- Middleware `jwt_required()` en todos los endpoints protegidos.
