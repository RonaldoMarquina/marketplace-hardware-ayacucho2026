# HU-06: Carga de Imágenes y Videos del Anuncio

**Como** vendedor autenticado
**Quiero** subir fotos o videos del componente
**Para** que compradores vean su estado físico.

## Endpoint

POST `/api/v1/anuncios/:anuncio_id/media`
🔒 JWT requerido | Content-Type: `multipart/form-data`

## Restricciones de archivo

| Tipo   | Formatos          | Tamaño máx | Cantidad máx por anuncio |
|--------|-------------------|------------|--------------------------|
| Imagen | JPG, PNG, WEBP    | 10MB       | 8                        |
| Video  | MP4               | 40MB       | 1                        |

- Validar mimetype real, no solo extensión.
- Un anuncio puede tener hasta 5 imágenes + 1 video simultáneamente.
- No se permiten GIF.

## Flujo

- Verificar JWT.
- Verificar anuncio existe en BD (sino 404).
- Verificar anuncio pertenece al usuario del JWT (sino 403).
- Separar archivos recibidos en dos grupos: imágenes y videos.
- Verificar que el anuncio no supere el límite de cada grupo por separado (sino 409).
- Procesar con Flask (Werkzeug `FileStorage`).
- Validar mimetype real de cada archivo antes de guardar.
- Renombrar: `{uuid}-{timestamp}.{ext}`.
- Crear carpeta `/uploads/anuncios/{anuncio_id}/` si no existe.
- Guardar archivo en disco (no en RAM).
- INSERT en `media_anuncio` con `tipo_media` = `imagen` o `video`.
- Si falla INSERT → eliminar archivo físico (rollback manual).
- Primera imagen recibida sin imagen principal previa: `es_principal=True`, `orden=0`.
- Imágenes siguientes: `orden=1, 2, 3...`
- Video: `es_principal=False`, `orden=null`.

## Respuesta

**201 Created**
```json
{
  "media": [
    { "id": 1, "tipo_media": "imagen", "ruta_relativa": "/uploads/anuncios/15/uuid-ts.jpg", "es_principal": true, "orden": 0 },
    { "id": 2, "tipo_media": "imagen", "ruta_relativa": "/uploads/anuncios/15/uuid-ts.jpg", "es_principal": false, "orden": 1 },
    { "id": 3, "tipo_media": "video", "ruta_relativa": "/uploads/anuncios/15/uuid-ts.mp4", "es_principal": false, "orden": null }
  ]
}
```

## Errores

- 400 Sin archivos adjuntos.
- 403 Anuncio no pertenece al usuario.
- 404 Anuncio no existe.
- 409 Anuncio ya tiene 8 imágenes / ya tiene 1 video.
- 413 Imagen supera 10MB / video supera 40MB.
- 422 Tipo de archivo inválido / más de 8 imágenes / más de 1 video en la petición.

## Reglas

- Nombre original nunca se guarda → siempre `{uuid}-{timestamp}.{ext}`.
- Ruta en BD es relativa, no absoluta.
- `/uploads/` en `.gitignore`.
- `tipo_media` en BD acepta solo los valores `imagen` o `video`.
- El límite de imágenes y el de video son independientes: subir el video no ocupa slot de imagen ni viceversa.
- Un anuncio solo puede tener 1 video; subir uno nuevo cuando ya existe → 409.
- `orden` aplica solo a imágenes; para videos siempre es `null`.
- El video no puede ser `es_principal`.
