# HU-06: Carga de Imágenes del Anuncio

**Como** vendedor autenticado  
**Quiero** subir fotos del componente  
**Para** que compradores vean su estado físico.

## Endpoint

POST `/api/v1/anuncios/:anuncio_id/imagenes`  
🔒 JWT requerido | Content-Type: `multipart/form-data`

## Restricciones de archivo

- Formatos: JPG, PNG, WEBP (no GIF)
- Tamaño: máx 2MB por imagen
- Cantidad: máx 5 por petición / 5 por anuncio
- Validar mimetype real, no solo extensión

## Flujo

- Verificar JWT.
- Verificar anuncio existe en BD (sino 404).
- Verificar anuncio pertenece al usuario del JWT (sino 403).
- Verificar anuncio tiene < 5 imágenes en BD (sino 409).
- Procesar con Flask (Werkzeug `FileStorage`).
- Renombrar: `{uuid}-{timestamp}.{ext}`.
- Crear carpeta `/uploads/anuncios/{anuncio_id}/` si no existe.
- Guardar archivo en disco (no en RAM).
- INSERT en `imagenes_anuncio`.
- Si falla INSERT → eliminar archivo físico (rollback manual).
- Primera imagen: `es_principal=True`, `orden=0`.
- Siguientes: `orden=1, 2, 3...`

## Respuesta

**201 Created**
```json
{
  "imagenes": [
    { "id": 1, "ruta_relativa": "/uploads/anuncios/15/uuid-ts.jpg", "es_principal": true, "orden": 0 },
    { "id": 2, "ruta_relativa": "/uploads/anuncios/15/uuid-ts.jpg", "es_principal": false, "orden": 1 }
  ]
}
```

## Errores

- 400 Sin archivos adjuntos.
- 403 Anuncio no pertenece al usuario.
- 404 Anuncio no existe.
- 409 Anuncio ya tiene 5 imágenes.
- 413 Imagen supera 2MB.
- 422 Tipo de archivo inválido / más de 5 archivos.

## Reglas

- Nombre original nunca se guarda → siempre `{uuid}-{timestamp}.{ext}`.
- Ruta en BD es relativa, no absoluta.
- `/uploads/` en `.gitignore`.
