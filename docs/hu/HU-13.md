## Flujo вЂ” Reportar (POST)

1. Verificar JWT + cuenta ACTIVO.
2. Verificar no es propietario del anuncio (sino 409).
3. Verificar no existe reporte PENDIENTE del mismo usuario (sino 409).
4. Validar `motivo` en ENUM con Marshmallow.
5. INSERT en `reportes` con estado=PENDIENTE.
6. **No cambiar estado del anuncio** вЂ” decisiГіn del admin.

## Flujo вЂ” Bloquear (PATCH)

Ejecutar en **transacciГіn SQLAlchemy**:
1. Verificar anuncio existe y estado=ACTIVO (sino 404/409).
2. Validar `motivo_admin` no vacГ­o (sino 400).
3. `UPDATE anuncios SET estado="BLOQUEADO"`.
4. `UPDATE reportes SET estado="REVISADO" WHERE anuncio_id AND estado="PENDIENTE"`.
5. `INSERT INTO moderacion_log (accion="BLOQUEADO")`.
6. Si falla cualquier paso в†’ ROLLBACK.

## Flujo вЂ” Desbloquear (PATCH)

Ejecutar en **transacciГіn SQLAlchemy**:
1. Verificar anuncio existe y estado=BLOQUEADO (sino 409).
2. `UPDATE anuncios SET estado="ACTIVO"`.
3. `INSERT INTO moderacion_log (accion="DESBLOQUEADO")`.
4. Reportes anteriores quedan como estado="REVISADO" (no eliminar).
