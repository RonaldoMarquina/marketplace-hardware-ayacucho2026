# HU-12: Contacto Directo por WhatsApp

## Objetivo

- Actor: comprador interesado
- Capacidad: obtener un enlace listo para contactar al vendedor
- Resultado esperado: iniciar conversacion por WhatsApp sin copiar datos
  manualmente

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `GET` | `/api/v1/anuncios/:anuncio_id/contacto` | Generar enlace de contacto |

El endpoint requiere JWT valido.

## Salida funcional

Se construye una URL con formato:

```text
https://wa.me/51{telefono}?text={mensaje_codificado}
```

## Flujo principal

1. validar `anuncio_id`
2. validar JWT
3. verificar estado habilitado del comprador
4. verificar existencia del anuncio y `estado = ACTIVO`
5. impedir autocontacto
6. verificar disponibilidad de telefono del vendedor
7. aplicar limite diario y limite por anuncio
8. sanitizar titulo del anuncio
9. construir mensaje
10. codificar el mensaje para URL
11. retornar el enlace generado
12. registrar contacto en `contactos_log`

## Datos relevantes

### Tabla implicada

- `contactos_log`

## Respuesta esperada

`200 OK` con:

- `whatsapp_url`
- `vendedor_nombre`
- `anuncio_titulo`

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | identificador invalido |
| `401` | sin JWT o token invalido |
| `403` | cuenta no habilitada para contactar |
| `404` | anuncio inexistente o no disponible |
| `409` | autocontacto |
| `422` | vendedor sin telefono |
| `429` | limite diario o limite por anuncio alcanzado |

## Reglas

- el telefono no se expone como dato aislado, solo dentro del enlace
- el prefijo `51` lo agrega el backend
- el contacto debe quedar trazado para futuras validaciones y metricas
