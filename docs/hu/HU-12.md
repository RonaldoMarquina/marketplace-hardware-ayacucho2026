# HU-12: Contacto Directo por WhatsApp

**Como** comprador interesado  
**Quiero** contactar al vendedor con un clic  
**Para** negociar sin copiar números manualmente.

## Endpoint

GET `/api/v1/anuncios/:anuncio_id/contacto`  
🔒 JWT requerido

## URL generada

```
https://wa.me/51{telefono}?text={encodeURIComponent(mensaje)}
```
Mensaje: `"Hola, vi tu anuncio '{titulo}' en HardwareAyacucho y me interesa. ¿Sigue disponible?"`

## Flujo (orden obligatorio)

1. Validar `anuncio_id` entero positivo (sino 400).
2. Verificar JWT + estado cuenta ACTIVO (sino 401/403).
3. Verificar anuncio existe y estado=ACTIVO (sino 404).
4. Verificar comprador ≠ vendedor (sino 409).
5. Verificar `telefono` del vendedor no es NULL (sino 422).
6. Construir URL: prefijo `51` + teléfono + mensaje codificado.
7. Retornar 200.
8. INSERT en `contactos_log` de forma **async sin await** (fallo no bloquea respuesta).

## Respuesta

**200 OK**
```json
{
  "whatsapp_url": "https://wa.me/51987654321?text=Hola%2C%20vi%20tu%20anuncio...",
  "vendedor_nombre": "Juan Quispe",
  "anuncio_titulo": "Ryzen 5 5600X casi nuevo"
}
```

## Errores

- 400 `anuncio_id` no es entero positivo.
- 401 Sin JWT o token inválido.
- 403 Cuenta PENDIENTE_VERIFICACION.
- 404 Anuncio inexistente o no ACTIVO.
- 409 Autocontacto (comprador == vendedor).
- 422 Vendedor sin teléfono registrado.

## Reglas

- Prefijo `51` siempre concatenado en backend, nunca en frontend.
- Teléfono nunca expuesto directamente, solo dentro de la URL.
- `encodeURIComponent()` al mensaje completo (tildes, `¿`, `?`).
- Log: `comprador_id, anuncio_id, created_at` — fallo de INSERT no afecta respuesta.
- Sin límite de contactos por usuario en este MVP.
