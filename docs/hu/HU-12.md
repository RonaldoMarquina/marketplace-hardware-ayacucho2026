# HU-12: Contacto Directo por WhatsApp

**Como** comprador interesado
**Quiero** contactar al vendedor con un clic
**Para** negociar sin copiar números manualmente.

## Endpoint

GET `/api/v1/anuncios/:anuncio_id/contacto`
🔒 JWT requerido

## URL generada

https://wa.me/51{telefono}?text={encodeURIComponent(mensaje)}

Mensaje: `Hola, vi tu anuncio "{titulo_sanitizado}" en HardwareAyacucho y me interesa. ¿Sigue disponible?`

- `titulo_sanitizado`: strip de comillas simples, dobles y caracteres de control antes de interpolar.
- `encodeURIComponent()` aplicado al mensaje completo después de sanitizar.

## Flujo (orden obligatorio)

1. Validar `anuncio_id` entero positivo (sino 400).
2. Verificar JWT válido (sino 401).
3. Verificar estado cuenta comprador:
   - `ACTIVO` → continuar.
   - `PENDIENTE_VERIFICACION` o `EN_REVISION` → 403.
   - `BLOQUEADO` → 403.
4. Verificar anuncio existe y `estado=ACTIVO` (sino 404).
5. Verificar comprador ≠ vendedor (sino 409).
6. Verificar `telefono` del vendedor no es NULL (sino 422).
7. Consultar `contactos_log` para rate limiting:
   - Si comprador generó ≥ 20 contactos distintos hoy → 429.
   - Si comprador contactó este anuncio específico en la última hora → 429.
8. Sanitizar `titulo` del anuncio.
9. Construir mensaje y aplicar `encodeURIComponent()`.
10. Construir URL: `https://wa.me/51{telefono}?text={mensaje_codificado}`.
11. Retornar 200.
12. INSERT en `contactos_log` de forma **async sin await** (fallo no bloquea respuesta).

## Respuesta

**200 OK**
```json
{
  "whatsapp_url": "https://wa.me/51987654321?text=Hola%2C%20vi%20tu%20anuncio%20%22Ryzen%205%205600X%22%20en%20HardwareAyacucho%20y%20me%20interesa.%20%C2%BFSigue%20disponible%3F",
  "vendedor_nombre": "Juan Quispe",
  "anuncio_titulo": "Ryzen 5 5600X casi nuevo"
}
```

**429 Too Many Requests — límite diario**
```json
{
  "error": "RATE_LIMIT_DIARIO",
  "mensaje": "Alcanzaste el límite de 20 contactos por día.",
  "disponible_en": "2026-06-21T00:00:00"
}
```

**429 Too Many Requests — mismo anuncio**
```json
{
  "error": "RATE_LIMIT_ANUNCIO",
  "mensaje": "Ya contactaste este anuncio recientemente. Intenta en 1 hora.",
  "disponible_en": "2026-06-20T15:30:00"
}
```

## Errores

- 400 `anuncio_id` no es entero positivo.
- 401 Sin JWT o token inválido.
- 403 Cuenta `PENDIENTE_VERIFICACION` / `EN_REVISION` / `BLOQUEADO`.
- 404 Anuncio inexistente / no `ACTIVO` (mismo mensaje, no revelar estado real).
- 409 Autocontacto (comprador == vendedor).
- 422 Vendedor sin teléfono registrado.
- 429 Límite diario de 20 contactos distintos alcanzado / mismo anuncio contactado en última hora.

## Esquema `contactos_log`

comprador_id   → FK usuarios
vendedor_id    → FK usuarios
anuncio_id     → FK anuncios
created_at     → TIMESTAMP

## Reglas

- Prefijo `51` siempre concatenado en backend, nunca en frontend.
- Teléfono nunca expuesto directamente — solo dentro de la URL construida.
- `titulo` sanitizado antes de interpolar: strip de `"`, `'` y caracteres de control.
- `encodeURIComponent()` aplicado al mensaje completo tras sanitizar.
- Rate limiting consultado desde `contactos_log` — sin tabla extra, sin Redis en MVP.
- Límite diario: 20 contactos a vendedores distintos por comprador por día calendario.
- Límite por anuncio: 1 contacto por hora por comprador al mismo anuncio.
- `disponible_en` en 429: medianoche local para límite diario, hora exacta de disponibilidad para límite por anuncio.
- Log guarda `vendedor_id` para métricas futuras de vendedores más contactados.
- INSERT en log async sin await — fallo de log nunca bloquea ni altera la respuesta.
- Anuncio `INACTIVO`, `VENDIDO` o `BLOQUEADO` → 404 aunque el comprador tenga el link directo.
- Sin límite de contactos para `TIENDA_VERIFICADA` como comprador — reservado para fases futuras.
