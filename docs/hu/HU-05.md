# HU-05: Publicar Anuncio de Hardware

## Objetivo

- Actor: vendedor autenticado y verificado
- Capacidad: registrar un anuncio con datos comerciales y especificaciones
  tecnicas
- Resultado esperado: publicar un producto visible para compradores

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `POST` | `/api/v1/anuncios` | Crear anuncio |

El endpoint requiere JWT valido y cuenta habilitada para publicar.

## Entrada

### Campos base

- `titulo`
- `categoria`
- `subcategoria`
- `condicion`
- `precio`
- `descripcion`
- `especificaciones` opcional

### Enumeraciones

#### Categoria

- `COMPONENTES`
- `REFRIGERACION`
- `GABINETES`
- `PERIFERICOS`
- `MONITORES`
- `REDES`
- `MOBILIARIO`
- `ALMACENAMIENTO_EXTERNO`
- `ACCESORIOS`
- `PORTATILES`

#### Condicion

- `NUEVO`
- `COMO_NUEVO`
- `USADO`
- `PARA_REPUESTOS`

## Validaciones funcionales

- `titulo` con longitud controlada
- `precio` decimal positivo
- `subcategoria` obligatoria
- `subcategoria` compatible con `categoria`
- `especificaciones` en formato JSON
- el JSON de especificaciones no debe duplicar campos clasificatorios como
  `categoria`, `subcategoria` o equivalentes

## Flujo principal

1. validar JWT
2. obtener `usuario_id` desde el token
3. verificar que la cuenta del usuario este habilitada para publicar
4. validar estructura del body
5. validar compatibilidad entre `categoria` y `subcategoria`
6. validar limite de anuncios activos segun rol
7. crear anuncio con `estado = ACTIVO`
8. inicializar `reactivaciones_count = 0`
9. registrar `created_at` y `updated_at`

## Respuesta esperada

`201 Created` con identificador del anuncio, datos principales y `estado =
ACTIVO`.

## Datos relevantes

### Politica de publicacion por rol

- `USER_ESTANDAR`: maximo `25` anuncios activos
- `TIENDA_VERIFICADA`: sin limite de anuncios activos

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | campos obligatorios ausentes |
| `401` | sin JWT o token invalido |
| `403` | cuenta no habilitada o limite de anuncios alcanzado |
| `422` | categoria, subcategoria, condicion, precio o especificaciones invalidas |

## Reglas

- `usuario_id` se toma exclusivamente del JWT
- la validacion de categoria y subcategoria se resuelve antes de persistir
- el anuncio nace como `ACTIVO`
- la moderacion no bloquea la publicacion inicial en el alcance actual
- el anuncio debe ser compatible con la maquina de estados definida en HU-08
