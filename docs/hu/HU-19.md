# HU-19: Panel del Usuario

**Como** usuario autenticado
**Quiero** ver y gestionar toda mi actividad en la plataforma
**Para** administrar mis anuncios, ventas, compras y reputación desde un solo lugar.

## Endpoint

GET `/api/v1/usuarios/me/panel`
🔒 JWT requerido

## Flujo

1. Verificar JWT válido + cuenta `ACTIVO` (sino 401/403).
2. Extraer `usuario_id` del JWT.
3. Ejecutar queries en paralelo (`asyncio.gather`):
   - Query perfil: nombre, correo, rol, estado, tienda (si aplica).
   - Query anuncios: agrupados por estado (`ACTIVO`, `INACTIVO`, `VENDIDO`).
   - Query reputación vendedor: promedio, total calificaciones, total ventas.
   - Query reputación comprador: promedio, total calificaciones, total compras.
   - Query calificaciones pendientes: count de transacciones con calificación pendiente.
4. Transformar rol a `es_tienda_verificada`.
5. Retornar panel consolidado.

## Errores

- 401 Sin JWT o token inválido.
- 403 Cuenta no `ACTIVO`.

## Reglas

- Endpoint privado — cada usuario solo ve su propio panel.
- `usuario_id` siempre del JWT, nunca del path ni del body.
- `correo` y `telefono` se exponen aquí — es el propio usuario viendo su información privada.
- `tienda.ruc` se expone aquí — es el propio usuario viendo su información privada, a diferencia de HU-17 donde es público.
- `estado` se expone aquí — es el propio usuario viendo su información, no aplica regla de transformación pública.
- `es_tienda_verificada` reemplaza `rol` crudo — consistente con todas las HUs.
- `anuncios.activos.limite_maximo`:
  - `USER_ESTANDAR` → `25`.
  - `TIENDA_VERIFICADA` → `null` (sin límite).
- `anuncios.activos.disponibles`:
  - `USER_ESTANDAR` → `25 - total_activos`.
  - `TIENDA_VERIFICADA` → `null`.
- `reputacion_vendedor` y `reputacion_comprador` son independientes — nunca mezclar en un solo promedio.
- `calificacion_promedio` con 1 decimal: `4.8`, nunca `4.833333`.
- `calificacion_promedio` es `null` si no tiene calificaciones — nunca `0`.
- `calificaciones_pendientes` cuenta todas las transacciones donde el usuario debe calificar — tanto como vendedor (HU-16) como comprador (HU-15).
- Este endpoint es solo lectura — las acciones (editar perfil, gestionar anuncios) van en sus respectivos endpoints.
- Queries en paralelo para minimizar latencia.
- Índices recomendados: `anuncios(usuario_id, estado)`, `transacciones(vendedor_id)`, `transacciones(comprador_id)`.
