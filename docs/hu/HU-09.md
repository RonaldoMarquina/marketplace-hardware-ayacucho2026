# HU-09: Feed Principal de Anuncios

**Como** visitante o usuario autenticado
**Quiero** ver anuncios recientes paginados
**Para** explorar ofertas sin necesidad de registro.

## Endpoint

GET `/api/v1/anuncios`
🌐 Público — sin JWT

## Query params

| Param | Default | Máx |
|-------|---------|-----|
| page  | 1       | —   |
| limit | 20      | 50  |

## Flujo

- Validar `page` (entero > 0) y `limit` (≤ 50).
- OFFSET = (page - 1) * limit.
- Query con `WHERE estado='ACTIVO' ORDER BY created_at DESC`.
- INNER JOIN usuarios → obtener nombre y rol del vendedor.
- Transformar rol internamente: `USER_ESTANDAR` → `false`, `TIENDA_VERIFICADA` → `true`.
- Subquery imagen_principal (`es_principal=True`).
- Calcular `total_paginas = ceil(total / limit)`.

## Respuesta

**200 OK**
```json
{
  "data": [
```

## Errores

- 400 `page` inválido (negativo, no numérico).
- 400 `limit` supera 50.

## Reglas

- Sin resultados → 200 con `data: []`, nunca 404.
- Página fuera de rango → 200 con `data: []` y `total` real.
- Campos excluidos: `especificaciones`, `usuario_id`, `telefono`, `estado`, `rol`.
- `imagen_principal` puede ser null → frontend debe manejarlo.
- `updated_at` puede ser null si el anuncio nunca fue editado → frontend debe manejarlo.
- ENUMs internos nunca salen crudos en respuestas públicas — siempre transformar antes de serializar.
- Sin caché, consulta en tiempo real.
- Índice compuesto en BD: `(estado, created_at)`.
