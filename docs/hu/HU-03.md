# HU-03: Verificación de Correo Electrónico

**Como** usuario recién registrado  
**Quiero** recibir un correo con enlace seguro  
**Para** activar mi cuenta.

## Endpoints

GET `/api/v1/auth/verify-email?token=XXXXXX`  
POST `/api/v1/auth/verify-email/resend`

## Flujo — Envío (post-registro)

- Al completar HU-01/HU-02, generar token con `secrets.token_hex(32)`.
- Guardar en `tokens_verificacion`: usuario_id, token, tipo="EMAIL_VERIFICATION", expira_en=NOW()+24h, usado=False.
- Enviar correo con enlace `https://[dominio_frontend]/verificar?token=...`.
- Si falla el envío → loggear error, NO revertir el registro.

## Flujo — Verificación (GET)

- Buscar token en BD.
- Validar: existe → no usado → no expirado.
- UPDATE usuarios SET estado="ACTIVO".
- UPDATE token SET usado=True.

## Flujo — Reenvío (POST)

- Validar que cuenta esté en PENDIENTE_VERIFICACION.
- Invalidar tokens anteriores del usuario (SET usado=True).
- Generar nuevo token, insertar con expira_en=NOW()+24h.
- Enviar correo.
- Rate limit: máx 3 req por IP/usuario cada 15 min.

## Respuestas

- 200 Verificación exitosa / correo reenviado.
- 404 Token inválido o inexistente.
- 409 Token ya usado / cuenta ya activa.
- 410 Token expirado.

## Reglas

- Token con `secrets.token_hex(32)`, vigencia 24h, un solo uso.
- Solo un token activo por usuario por tipo.
- Estado PENDIENTE_VERIFICACION: no puede login ni publicar.
- Envío de correo async con try/except independiente (no bloquea).
- Índice en columna `token` en BD.
