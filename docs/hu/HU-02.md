# HU-02: Registro de Tienda Verificada

**Como** propietario de tienda de hardware en Perú  
**Quiero** registrar mi negocio con datos comerciales  
**Para** obtener un perfil verificado que genere confianza.

## Endpoint

POST `/api/v1/auth/register/tienda`  
Content-Type: `multipart/form-data`

## Entrada

- nombre_comercial
- ruc (11 dígitos)
- direccion
- telefono (formato 9 digitos)
- correo
- password (deve contener al menos un caracter especial, una Letra mayuscula y un numeros)
- documento_identidad (JPG/PNG, max 5MB)

## Flujo

- Validar con Marshmallow.
- Validar archivo con Flask (JPG/PNG, ≤5MB).
- Verificar nombre_comercial unico en tiendas.
- Verificar correo único en usuarios.
- Verificar RUC único en tiendas.
- Verificar  telefono unico en ususarios. 
- Cifrar con bcrypt (salt=10).
- Insertar en usuarios y tiendas dentro de una transacción SQLAlchemy.
- Si falla algún INSERT → rollback.
- Guardar documento como `uuid_documento.ext`.
- Rol: TIENDA_VERIFICADA. Estado: EN_REVISION.

## Respuesta

**201 Created**

```json
{
  "id": 1,
  "nombre_comercial": "TechStore Ayacucho",
  "ruc": "20601234567",
  "correo": "tienda@gmail.com",
  "rol": "TIENDA_VERIFICADA",
  "estado": "EN_REVISION"
}
```

## Errores

- 400 Campos inválidos / documento no adjuntado.
- 409 Correo duplicado / RUC duplicado / nombre_comercial duplicado / telefono duplicado.
- 415 Formato de archivo no permitido.
- 422 RUC con formato inválido / formato de telefono no valido.

## Reglas

- RUC único (constraint BD), exactamente 11 dígitos.
- Transacción BD: rollback si falla algún INSERT.
- Documento guardado con nombre UUID, no el original.
- Estado EN_REVISION: no puede publicar anuncios (403).
- Distintivo "Tienda Verificada" solo cuando estado=ACTIVO.
- Admin cambia estado manualmente: EN_REVISION → ACTIVO o RECHAZADO.
- Nunca exponer password.
