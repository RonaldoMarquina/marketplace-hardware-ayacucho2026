# HU-11: Vista Detallada del Anuncio

**Como** comprador interesado
**Quiero** ver toda la info técnica, fotos y videos de un anuncio
**Para** decidir si contactar al vendedor.

## Endpoint

GET `/api/v1/anuncios/:anuncio_id`
🌐 Público (JWT opcional — desbloquea teléfono)

## Flujo

1. Validar `anuncio_id` es entero positivo (sino 400).
2. Ejecutar ambas queries en paralelo (`asyncio.gather`):
   - Query principal: anuncio + vendedor + tienda (LEFT JOIN).
   - Query media: `ORDER BY tipo_media ASC, orden ASC` (imágenes primero, video al final).
3. Si no existe o estado ∉ {ACTIVO} → verificar JWT:
   - Sin JWT → 404.
   - Con JWT → verificar si es propietario:
     - Es propietario + estado == INACTIVO → devolver anuncio con `es_propietario: true`.
     - Es propietario + estado == VENDIDO o BLOQUEADO → 404.
     - No es propietario → 404.
4. Transformar rol internamente:
   - `USER_ESTANDAR` → `es_tienda_verificada: false`, omitir objeto `tienda`.
   - `TIENDA_VERIFICADA` → `es_tienda_verificada: true`, incluir objeto `tienda`.
5. Según JWT para teléfono:
   - Sin JWT → `telefono: null`.
   - Con JWT válido → `telefono: "987654321"`.
6. Determinar `es_propietario`:
   - Sin JWT → `false`.
   - Con JWT → comparar `usuario_id` del JWT con `anuncio.usuario_id`.
7. Log: anuncio_id, timestamp, autenticado/visitante.

## Errores

- 400 `anuncio_id` no es entero positivo.
- 404 Anuncio inexistente / VENDIDO / BLOQUEADO / INACTIVO sin ser propietario (mismo mensaje genérico, nunca revelar estado real).

## Reglas

- `estado` nunca se expone directamente en la respuesta — para terceros simplemente no existe, para el propietario se encapsula en `estado_propietario`.
- `es_propietario` siempre presente: `true` o `false`, nunca omitir.
- `estado_propietario` solo aparece cuando `es_propietario: true`.
- `estado_propietario.reactivaciones_restantes` = 3 - `reactivaciones_count` del anuncio.
- `vendedor.telefono` siempre presente: `null` o valor real, nunca omitir el campo.
- `vendedor.tienda` aparece solo cuando `es_tienda_verificada: true` — independiente del JWT.
- `es_tienda_verificada` reemplaza `rol` — ENUMs internos nunca salen crudos.
- `especificaciones` como objeto JSON, nunca string.
- `media: []` si no hay archivos, nunca 404.
- `updated_at` puede ser null si nunca fue editado → frontend debe manejarlo.
- Imágenes ordenadas por `orden ASC`, video siempre al final (`orden: null`).
- Propietario autenticado puede ver sus INACTIVOS; VENDIDO y BLOQUEADO → 404 incluso para propietario.
- Sin contador de vistas en MVP, solo log.
- Campos excluidos: `usuario_id`, `telefono` del body principal, `rol`.
