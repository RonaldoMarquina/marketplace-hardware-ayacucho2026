# HU-11: Vista Detallada del Anuncio

## Objetivo

- Actor: comprador interesado
- Capacidad: consultar informacion completa de un anuncio y de su vendedor
- Resultado esperado: evaluar el producto antes de iniciar contacto

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `GET` | `/api/v1/anuncios/:anuncio_id` | Obtener detalle del anuncio |

El endpoint es publico. El JWT es opcional y habilita informacion adicional
segun contexto.

## Flujo principal

1. validar `anuncio_id`
2. consultar anuncio, vendedor y media asociada
3. si el anuncio no esta disponible publicamente, evaluar si el solicitante es
   el propietario y si el estado permite vista privada
4. transformar `rol` a `es_tienda_verificada`
5. incluir informacion de tienda solo cuando aplique
6. exponer telefono segun autenticacion valida
7. calcular `es_propietario`
8. devolver detalle consolidado

## Visibilidad por contexto

- terceros solo pueden ver anuncios `ACTIVO`
- el propietario puede ver anuncios `INACTIVO`
- anuncios `VENDIDO` o `BLOQUEADO` no deben exponerse por este endpoint
- sin JWT, el telefono del vendedor debe ser `null`

## Respuesta esperada

`200 OK` con:

- datos del anuncio
- `especificaciones`
- `media`
- datos del vendedor
- `es_propietario`
- `estado_propietario` solo cuando corresponda

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | identificador invalido |
| `404` | anuncio inexistente o no visible para el contexto actual |

## Reglas

- `estado` no se expone directamente a terceros
- `estado_propietario` solo aparece para el propietario en contexto permitido
- `telefono` siempre debe existir en la respuesta del vendedor, aunque sea
  `null`
- `media` debe mantener orden estable: imagenes primero y video al final
- no se exponen `rol`, `usuario_id` ni campos internos innecesarios
