# HU-11: Vista Detallada del Anuncio

**Como** comprador interesado  
**Quiero** ver toda la info técnica y fotos de un anuncio  
**Para** decidir si contactar al vendedor.

## Endpoint

GET `/api/v1/anuncios/:anuncio_id`  
🌐 Público (JWT opcional — desbloquea teléfono)

## Flujo

1. Validar `anuncio_id` es entero positivo (sino 400).
2. Ejecutar ambas queries en paralelo (`asyncio.gather`):
   - Query principal: anuncio + vendedor + tienda (LEFT JOIN).
   - Query imágenes: `ORDER BY orden ASC`.
3. Si no existe o estado ≠ ACTIVO → 404.
   - Excepción: propietario autenticado puede ver sus INACTIVOS.
4. Según JWT:
   - Sin JWT → `vendedor.telefono = null`.
   - Con JWT válido → `vendedor.telefono = "987654321"`.
5. Si vendedor es TIENDA_VERIFICADA → incluir objeto `tienda`.
6. Si vendedor es USER_ESTANDAR → omitir objeto `tienda`.
7. Log: anuncio_id, timestamp, autenticado/visitante.

## Respuesta

**200 OK — visitante**
```json
{
  "id": 15,
  "titulo": "Ryzen 5 5600X casi nuevo",
  "descripcion": "...",
  "categoria": "PROCESADOR",
  "condicion": "COMO_NUEVO",
  "precio": 450.00,
  "especificaciones": { "socket": "AM4", "nucleos": 6 },
  "estado": "ACTIVO",
  "created_at": "2026-06-20T14:30:00",
  "updated_at": "2026-06-20T14:30:00",
  "imagenes": [
    { "id": 1, "ruta_relativa": "/uploads/anuncios/15/abc.jpg", "es_principal": true, "orden": 0 }
  ],
  "vendedor": {
    "id": 7,
    "nombre": "Juan Quispe",
    "rol": "USER_ESTANDAR",
    "telefono": null
  }
}
```

**200 OK — usuario autenticado + tienda**
```json
"vendedor": {
  "id": 3,
  "nombre": "Carlos Huamani",
  "rol": "TIENDA_VERIFICADA",
  "telefono": "987654321",
  "tienda": {
    "nombre_comercial": "TechStore Ayacucho",
    "direccion": "Jr. Lima 123, Ayacucho"
  }
}
```

**200 OK — propietario ve su INACTIVO**
```json
{ "estado": "INACTIVO", "es_propietario": true, ... }
```

## Errores

- 400 `anuncio_id` no es entero positivo.
- 404 Anuncio inexistente, VENDIDO o BLOQUEADO (mismo mensaje, no revelar estado).

## Reglas

- `vendedor.telefono` siempre presente: `null` o valor real (nunca omitir).
- `especificaciones` como objeto JSON, nunca string.
- `imagenes: []` si no hay imágenes, nunca 404.
- Propietario autenticado puede ver sus INACTIVOS; terceros reciben 404.
- Sin contador de vistas en MVP, solo log.
