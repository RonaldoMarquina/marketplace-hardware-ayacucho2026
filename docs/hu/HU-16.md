# HU-16: Calificar al Comprador

**Como** vendedor que concretó una venta
**Quiero** calificar al comprador y dejar un comentario
**Para** ayudar a otros vendedores a identificar compradores confiables.

## Endpoint

POST `/api/v1/transacciones/:transaccion_id/calificar/comprador`
🔒 JWT requerido

## Body

```json
{
  "puntaje": 5,
  "comentario": "Comprador serio, pagó rápido y sin problemas."
}
```

## Flujo (orden obligatorio)

1. Validar `transaccion_id` entero positivo (sino 400).
2. Verificar JWT válido + cuenta `ACTIVO` (sino 401/403).
3. Verificar transacción existe (sino 404).
4. Verificar que `usuario_id` del JWT == `transaccion.vendedor_id` (sino 403).
5. Verificar `calificacion_comprador_pending=TRUE` en `transacciones` (sino 409).
6. Validar `puntaje` entero entre 1 y 5 (sino 422).
7. Validar `comentario` opcional, max 500 chars, sanitizar antes de guardar (sino 422).
8. Ejecutar en **transacción SQLAlchemy**:
   - `INSERT INTO calificaciones (transaccion_id, calificador_id, calificado_id, tipo, puntaje, comentario, created_at)`.
   - `UPDATE transacciones SET calificacion_comprador_pending=FALSE`.
   - Recalcular `calificacion_promedio` y `total_calificaciones` en tabla `usuarios` para el comprador.
9. Log: `vendedor_id, comprador_id, transaccion_id, puntaje, timestamp`.

## Respuesta

**200 OK**
```json
{
  "calificacion_id": 32,
  "transaccion_id": 8,
  "calificado": {
    "id": 42,
    "nombre": "Pedro Flores",
    "es_tienda_verificada": false,
    "calificacion_promedio": 4.6,
    "total_calificaciones": 11
  },
  "puntaje": 5,
  "comentario": "Comprador serio, pagó rápido y sin problemas.",
  "created_at": "2026-06-20T21:30:00"
}
```

## Errores

- 400 `transaccion_id` inválido.
- 401 Sin JWT o token inválido.
- 403 No es el vendedor de la transacción / cuenta no ACTIVO.
- 404 Transacción no existe.
- 409 Comprador ya calificado en esta transacción.
- 422 `puntaje` fuera de rango 1–5 / `comentario` > 500 chars.

## Reglas

- Solo el vendedor de la transacción puede calificar al comprador — verificado desde `transaccion.vendedor_id`.
- Solo se puede calificar una vez por transacción — `calificacion_comprador_pending` como guardia.
- `puntaje` entero del 1 al 5, sin decimales — el promedio lo calcula el backend.
- `comentario` opcional pero sanitizado siempre — strip de HTML y caracteres de control.
- `calificacion_promedio` en tabla `usuarios` se recalcula en cada nueva calificación — nunca se guarda stale.
- `calificacion_promedio` expuesto con 1 decimal: `4.6`, no `4.666666`.
- Calificación visible públicamente en perfil del comprador (HU-17) inmediatamente.
- Calificación no editable ni eliminable por el usuario — solo admin puede intervenir.
- `calificaciones` es registro permanente — nunca se elimina salvo intervención admin.
- HU-15 y HU-16 son independientes — el comprador puede calificar al vendedor sin que el vendedor haya calificado al comprador y viceversa.
- Sin período de gracia para calificar en este MVP — se habilita al marcar VENDIDO y no expira.
- Comparte esquema `calificaciones` con HU-15 — mismo tabla, `tipo=VENDEDOR_A_COMPRADOR`.
