# HU-01: Registro de Usuario Estandar

## Objetivo

- Actor: ciudadano de Ayacucho
- Capacidad: crear una cuenta personal con correo y contrasena
- Resultado esperado: iniciar el flujo de acceso a la plataforma como usuario
  estandar

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `POST` | `/api/v1/auth/register` | Registrar usuario estandar |

## Entrada

- `nombre`
- `correo`
- `password`
- `telefono`

## Flujo principal

1. validar campos requeridos
2. validar formato de correo
3. validar complejidad de contrasena
4. validar formato de telefono
5. verificar unicidad de correo
6. verificar unicidad de telefono
7. generar hash de contrasena
8. crear usuario con `rol = USER_ESTANDAR`
9. asignar `estado = PENDIENTE_VERIFICACION`
10. disparar el flujo de verificacion de correo

## Respuesta esperada

`201 Created` con identificador, datos principales del usuario, `rol` y
`estado`.

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | campos obligatorios ausentes |
| `409` | correo o telefono duplicado |
| `422` | correo, telefono o contrasena invalida |

## Reglas

- la contrasena nunca se expone en la respuesta
- la cuenta no puede iniciar sesion ni publicar hasta completar HU-03
- el fallo en el envio del correo no debe revertir el alta del usuario
