# HU-17: Perfil Público de Usuario

**Como** comprador o visitante
**Quiero** ver el perfil público de un vendedor o comprador
**Para** evaluar su reputación antes de contactar o realizar una transacción.

## Endpoint

GET `/api/v1/usuarios/:usuario_id/perfil`
🌐 Público — sin JWT

## Flujo

1. Validar `usuario_id` entero positivo (sino 400).
2. Verificar usuario existe y `estado=ACTIVO` (sino 404).
3. Ejecutar queries en paralelo (`asyncio.gather`):
   - Query usuario + tienda (LEFT JOIN — solo si `TIENDA_VERIFICADA`).
   - Query reputación como vendedor: `calificacion_promedio_vendedor`, `total_ventas`, `total_calificaciones_vendedor`.
   - Query reputación como comprador: `calificacion_promedio_comprador`, `total_compras`, `total_calificaciones_comprador`.
   - Query anuncios activos: últimos 10 `ORDER BY created_at DESC`.
4. Transformar rol: `USER_ESTANDAR` → `es_tienda_verificada: false`, `TIENDA_VERIFICADA` → `es_tienda_verificada: true`.
5. Incluir objeto `tienda` solo si `es_tienda_verificada: true`.
6. Retornar perfil consolidado.

## Errores

- 400 `usuario_id` inválido.
- 404 Usuario no existe / `estado` ≠ `ACTIVO`.

## Reglas

- Endpoint público — sin JWT, sin autenticación requerida.
- Usuario `BLOQUEADO`, `EN_REVISION`, `PENDIENTE_VERIFICACION` → 404 — no revelar estado real.
- `reputacion_vendedor` y `reputacion_comprador` son independientes — nunca mezclar en un solo promedio.
- `calificacion_promedio` con 1 decimal: `4.8`, no `4.833333`.
- `calificacion_promedio` es `null` si no tiene calificaciones — nunca `0` (no es lo mismo sin calificaciones que tener promedio cero).
- `reputacion_vendedor` se calcula desde `calificaciones WHERE tipo="COMPRADOR_A_VENDEDOR"`.
- `reputacion_comprador` se calcula desde `calificaciones WHERE tipo="VENDEDOR_A_COMPRADOR"`.
- `anuncios_activos` muestra máximo 10 — para ver todos usar HU-10 filtrando por vendedor (futura mejora).
- Campos excluidos: `telefono`, `correo`, `password`, `rol`, `estado`, `intentos_fallidos`, `bloqueado_hasta`, `usuario_id`.
- `tienda.ruc` y `tienda.documento_identidad` excluidos — son datos privados del negocio.
- `tienda.direccion` y `tienda.nombre_comercial` son públicos — ayudan al comprador a ubicar la tienda física.
- ENUMs internos nunca salen crudos — `rol` se transforma a `es_tienda_verificada`.
- `miembro_desde` expone solo `created_at` del usuario — sin hora exacta si se prefiere privacidad (decisión de implementación).
- Queries en paralelo para minimizar latencia — perfil + reputación vendedor + reputación comprador + anuncios.
- Índices recomendados: `calificaciones(calificado_id, tipo)`, `anuncios(usuario_id, estado)`.
