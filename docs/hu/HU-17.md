# HU-17: Perfil Publico de Usuario

## Objetivo

- Actor: comprador o visitante
- Capacidad: consultar reputacion y actividad publica de un usuario
- Resultado esperado: evaluar confiabilidad antes de contactar o comprar

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `GET` | `/api/v1/usuarios/:usuario_id/perfil` | Obtener perfil publico |

El endpoint es publico.

## Flujo principal

1. validar `usuario_id`
2. verificar existencia del usuario
3. permitir el acceso solo a cuentas `ACTIVO`
4. consultar informacion base del usuario
5. unir datos de tienda cuando el perfil corresponda a una tienda verificada
6. consultar reputacion como vendedor
7. consultar reputacion como comprador
8. consultar anuncios activos recientes
9. transformar `rol` a `es_tienda_verificada`
10. devolver perfil consolidado

## Datos expuestos

- identidad publica basica
- condicion de tienda verificada
- fecha de antiguedad
- reputacion como vendedor
- reputacion como comprador
- anuncios activos recientes

## Datos excluidos

- correo
- telefono
- password
- rol interno
- estado interno
- datos privados de verificacion de tienda

## Respuesta esperada

`200 OK` con perfil publico consolidado.

## Errores

| Codigo | Situacion |
|--------|-----------|
| `400` | identificador invalido |
| `404` | usuario inexistente o no visible publicamente |

## Reglas

- usuarios no activos deben responder como no visibles
- la reputacion de vendedor y comprador se calcula por separado
- los promedios deben expresarse con precision controlada
- si no hay calificaciones, el promedio debe ser `null`
- `tienda` solo se incluye cuando el usuario es tienda verificada
