# HU-15: Calificar al Vendedor

**Como** comprador que concretó una compra
**Quiero** calificar al vendedor y dejar un comentario
**Para** ayudar a otros compradores y construir reputación en la plataforma.

## Endpoint

POST `/api/v1/transacciones/:transaccion_id/calificar/vendedor`
🔒 JWT requerido

## Body

```json
{
  "puntaje": 5,
  "comentario": "Excelente vendedor, producto tal como se describió."
}
```

## Flujo (orden obligatorio)

1. Validar `transaccion_id` entero positivo (sino 400).
2. Verificar JWT válido + cuenta `ACTIVO` (sino 401/403).
3. Verificar transacción existe (sino 404).
4. Verificar que `usuario_id` del JWT == `transaccion.comprador_id` (sino 403).
5. Verificar `calificacion_vendedor_pending=TRUE` en `transacciones` (sino 409).
6. Validar `puntaje` entero entre 1 y 5 (sino 422).
7. Validar `comentario` opcional, max 500 chars, sanitizar antes de guardar (sino 422).
8. Ejecutar en **transacción SQLAlchemy**:
   - `INSERT INTO calificaciones (transaccion_id, calificador_id, calificado_id, tipo, puntaje, comentario, created_at)`.
   - `UPDATE transacciones SET calificacion_vendedor_pending=FALSE`.
   - Recalcular `calificacion_promedio` y `total_calificaciones` en tabla `usuarios` para el vendedor.
9. Log: `comprador_id, vendedor_id, transaccion_id, puntaje, timestamp`.

## Respuesta

**200 OK**
```json
{
  "calificacion_id": 31,
  "transaccion_id": 8,
  "calificado": {
    "id": 7,
    "nombre": "Juan Quispe",
    "es_tienda_verificada": false,
    "calificacion_promedio": 4.8,
    "total_calificaciones": 24
  },
  "puntaje": 5,
  "comentario": "Excelente vendedor, producto tal como se describió.",
  "created_at": "2026-06-20T21:00:00"
}
```

## Errores

- 400 `transaccion_id` inválido.
- 401 Sin JWT o token inválido.
- 403 No es el comprador de la transacción / cuenta no ACTIVO.
- 404 Transacción no existe.
- 409 Vendedor ya calificado en esta transacción.
- 422 `puntaje` fuera de rango 1–5 / `comentario` > 500 chars.

## Esquema `calificaciones`

id              → PK
transaccion_id  → FK transacciones
calificador_id  → FK usuarios (quien califica)
calificado_id   → FK usuarios (quien recibe la calificación)
tipo            → ENUM: COMPRADOR_A_VENDEDOR | VENDEDOR_A_COMPRADOR
puntaje         → TINYINT 1–5
comentario      → TEXT nullable
created_at      → TIMESTAMP

## Reglas

- Solo el comprador de la transacción puede calificar al vendedor — verificado desde `transaccion.comprador_id`.
- Solo se puede calificar una vez por transacción — `calificacion_vendedor_pending` como guardia.
- `puntaje` entero del 1 al 5, sin decimales — el promedio lo calcula el backend.
- `comentario` opcional pero sanitizado siempre — strip de HTML y caracteres de control.
- `calificacion_promedio` en tabla `usuarios` se recalcula en cada nueva calificación — nunca se guarda stale.
- `calificacion_promedio` expuesto con 1 decimal: `4.8`, no `4.833333`.
- Calificación visible públicamente en perfil del vendedor (HU-17) inmediatamente.
- Calificación no editable ni eliminable por el usuario — solo admin puede intervenir.
- `calificaciones` es registro permanente — nunca se elimina salvo intervención admin.
- Si el vendedor también es comprador en otra transacción, sus calificaciones como vendedor y como comprador son independientes.
- Sin período de gracia para calificar en este MVP — se habilita al marcar VENDIDO y no expira.
