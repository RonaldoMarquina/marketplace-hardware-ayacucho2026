# HU-18: Historial de Transacciones del Usuario

## Objetivo

- Actor: usuario autenticado
- Capacidad: consultar compras y ventas realizadas en la plataforma
- Resultado esperado: hacer seguimiento de transacciones y calificaciones
  pendientes

## Interfaz

| Metodo | Ruta | Finalidad |
|--------|------|-----------|
| `GET` | `/api/v1/usuarios/me/transacciones` | Obtener historial del usuario |

El endpoint requiere JWT valido.

## Parametros

- `tipo`
- `page`
- `limit`

## Flujo principal

1. validar JWT y cuenta `ACTIVO`
2. obtener `usuario_id` desde el token
3. validar filtros de consulta
4. consultar transacciones donde el usuario sea vendedor, comprador o ambos
5. unir informacion del anuncio y de la contraparte
6. determinar estado de calificacion pendiente por cada transaccion
7. ordenar por fecha descendente
8. paginar resultados
9. construir resumen global del usuario

## Respuesta esperada

`200 OK` con:

- `data`
- `resumen`
- `paginacion`

## Errores

| Codigo | Situacion |
|--------|-----------|
| `401` | sin JWT o token invalido |
| `403` | cuenta no activa |
| `422` | filtro `tipo` invalido |

## Reglas

- el historial es estrictamente privado
- `usuario_id` se obtiene solo del JWT
- cada item debe indicar si la transaccion es compra o venta para el usuario
  autenticado
- la contraparte es siempre el otro participante de la transaccion
- los datos privados de la contraparte no deben exponerse
