# HU-07: Edición de Anuncio Propio

**Como** vendedor autenticado  
**Quiero** editar mis anuncios publicados  
**Para** corregir errores o actualizar precio/descripción.

## Endpoint

PATCH `/api/v1/anuncios/:anuncio_id`  
🔒 JWT requerido

## Campos editables

titulo | descripcion | condicion | precio | especificaciones

## Campos inmutables

categoria | usuario_id | estado | created_at  
→ Ignorar si llegan en el body (no retornar error salvo `categoria`)

## Flujo (orden obligatorio)

1. Validar JWT.
2. Verificar anuncio existe (sino 404).
3. Verificar propietario: `usuario_id` del JWT == `anuncio.usuario_id` (sino 403).
4. Verificar estado == ACTIVO (VENDIDO/INACTIVO → 409, BLOQUEADO → 403).
5. Verificar body no vacío (sino 400).
6. Rechazar si body contiene `categoria` (422).
7. Validar campos con Marshmallow.
8. UPDATE solo campos presentes en body + `updated_at=NOW()`.
9. Log: usuario_id, anuncio_id, campos modificados.

## Respuesta

**200 OK**
```json
{
  "id": 15,
  "titulo": "Ryzen 5 5600X",
  "categoria": "PROCESADOR",
  "condicion": "COMO_NUEVO",
  "precio": 380.00,
  "descripcion": "Precio rebajado, negociable",
  "estado": "ACTIVO",
  "updated_at": "2026-01-01T00:00:00"
}
```

## Errores

- 400 Body vacío.
- 403 No es propietario / anuncio BLOQUEADO.
- 404 Anuncio no existe.
- 409 Anuncio VENDIDO o INACTIVO.
- 422 Campo inmutable `categoria` / precio inválido / título > 100 chars / condicion fuera de ENUM.

## Reglas

- PATCH: solo actualizar campos presentes en body.
- `usuario_id` siempre del JWT, nunca del body.
- `especificaciones` debe ser JSON válido.
- `condicion` debe pertenecer al ENUM si se envía.
